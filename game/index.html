TankProvider - TankProviderInterface - ITankProvider

The Tank Provider Interface contains a single public function "RequestTank". The RequestTank function will always return a single instance of a Tank interface object.

public TankInterface requestTank(String ip, int port, String tankName)

Tank - TankInterface - ITank

The Tank contains the three main functions that can be used to interact with your Tank they are Move, Scan, and Fire. Each action hasa delay before its executed and after its executed before the results are returned to the requester.

Move

Move the tank one space in the specified direction.
The before delay is 200ms.
The after delay is 300ms.

public MoveActionData move(Direction d)
MoveActionData moveData = myTank.move(Direction.NORTH);
public MoveActionData Move(Direction d)
MoveActionData moveData = myTank.Move(Direction.EAST);

Scan

Detect the next closest object in each of the four directions. NOTE: No limitation on range.
The before delay is 0ms.
The after delay is 100ms.

public ScanActionData scan()
ScanActionData scanData = myTank.scan();
public ScanActionData Scan()
ScanActionData scanData = myTank.Scan();

Fire

Fire a bullet in the specified direction. NOTE: No limitation on range.
The before delay is 500ms.
The after delay is 200ms.

public FireActionData fire(Direction d)
FireActionData fireData = myTank.fire(Direction.SOUTH);
public FireActionData Fire(Direction d)
FireActionData fireData = myTank.Fire(Direction.WEST);

Direction

The Direction enum is used to depict the cardinal direction of actions or responses. North is the top of the screen.

Direction.NORTH == 0, Direction.EAST == 1, Direction.SOUTH == 2, Direction.WEST == 3
myTank.move(Direction.NORTH);
myTank.fire(Direction.EAST);
ScanActionData scanData;
scanData.result(Direction.SOUTH);
Direction.NORTH == 0, Direction.EAST == 1, Direction.SOUTH == 2, Direction.WEST == 3
myTank.Move(Direction.NORTH);
myTank.Fire(Direction.EAST);
ScanActionData scanData;
scanData.result(Direction.SOUTH);

ActionData and TankStatus

The ActionData object is the super class of all other ActionData types. It contains a TankStatus object with three members. Health, ammo, and score. Tanks have 100 health when they spawn. Every time you make a request a TankStatus is returned along with it.

public class ActionData {
public TankStatus status;
}
TankStatus myTankStatus = anyActionData.status;
if (myTankStatus.health <= 40) runAway();
if (myTankStatus.ammo <= 3) saveAmmo();
if (myTankStatus.score >= 0) winning();
public class ActionData {
public TankStatus status;

ScanActionData

The ScanActionData response contains a "result" field of type ScanData[] which gives the nearest entity as described by EntityType in each direction. You can index into the array by using the Direction enum or by calling the the result function.

public class ScanActionData extends ActionData {
public ScanData[] result;
}
ScanData[] scanResult = scanActionData.result;
ScanData northData = scanResult[Direction.NORTH];
// OR
ScanData southData = scanResult.result(Direction.NORTH);
public class ScanActionData : ActionData {
public ScanData[] result;
}
ScanData[] scanResult = scanActionData.result;
ScanData northData = scanResult[Direction.NORTH];
// OR
ScanData southData = scanResult.Result(Direction.NORTH);

ScanActionData.Result

Result retrieves a ScanData object for a particular Direction from the result array.

public ScanData result(Direction d);
ScanData scanData = myScanActionData.result(Direction.EAST);
public ScanData Result(Direction d);
ScanData scanData = myScanActionData.result(Direction.EAST);

ScanData

The ScanData object gives you information about what was detected in a particular direction.

ScanData data;
data.entityType; // The type of entity detected see below.
data.distance; // Number of spaces between tank and scanned entity.
data.direction // Direction that this result came from.

The EntityType enum is used to indicate what type of entity was detected from a Scan action.

EntityType.WALL, Direction.TANK

if (scanData.result(Direction.NORTH).entityType == EntityType.TANK) Shoot();

Exceptions

The TankProviderInterface function (requestTank()) as well as each of the TankInterface functions (move(), scan(), fire()) each can throw the following Runtime Exceptions:

  • TankDeadException: Is thrown when you make a request and the tank is dead.
    However its only thrown once the following request is blocked till the tank respawns and then is honoured.
  • TankGameException: Is thrown when anything else unexpected happens.
    You can catch this in order to reset and re-attemp actions with your tank.
    Or when the game has ended and you have tried to keep going.

The most important exception of these is the TankDeadException. It is a means of determining when your tank dies while making a request.

TankDeadException extends RuntimeException {...}
TankGameException extends RuntimeException{...}
try {
    myTank.move(Direction.EAST)
} catch (TankDeadException ex) {
    resetAI()
}
TankDeadException extends Exception {...}
TankGameException extends Exception {...}
try {
    myTank.Move(Direction.EAST)
} catch (TankDeadException ex) {
    resetAI()
}
